Instalación

Instalación de FacturaScripts en tu PC (también lo puedes instalar en un hosting). Y primeros pasos tras la instalación.

Para instalar FacturaScripts solamente necesitas un servidor web, PHP 5.3 o superior y MySQL o PostgreSQL. Si lo que buscas en un manual rápido para instalarlo en Linux, aquí lo tienes.

Primeros pasos

FacturaScripts genera al vuelo todo lo que va necesitando: almacenes, divisas, paises, series, formas de pago, impuestos, etc...

Lo primero que debes hacer es ir a Admin > Empresa e introducir los datos de tu empresa. Tu país, que pasará a seleccionarse por defecto, tu divisa (moneda), que será la que se use por defecto, la forma de pago, la serie, etc.

En la sección facturación encontrará el checkbox contabilidad integrada, si lo marcas, lo que sucede es que FacturaScripts generará automáticamente los asientos correspondientes a las facturas tanto de compras como de ventas. Así la parte de contabilidad se genera automáticamente y tú ahorras horas y horas de trabajo.

FacturaScripts es prácticamente un framework MVC escrito en PHP 5.3. Puede conectar tanto a MySQL como a PostgreSQL. La capa gráfica está construida sobre bootstrap para adaptarse a PC, tablet y móvil. Y su sistema de plugins permite modificar el comportamiento de cualquier clase.

¿Quieres crear un plugin?

Pues simplemente crea una carpeta dentro de la carpeta plugins de FacturaScripts.

Si ahora vas al menú Admin > Panel de control, en la pestaña plugins verás que aparece un nuevo plugin con el nombre de la carpeta. Pero no puedes activarlo, porque le falta el archivo facturascripts.ini

  facturascripts.ini

Este archivo sirve para darle a FacturaScripts información importante sobre tu plugin: versión, url para desdecargar actualizaciones, etc. 

version = 1
; Indica la versión del plugin, tiene que ser un número entero.

version_url = 'http...'
; opcional Una URL donde consultar este archivo facturascripts.ini actualizado, así el actualizador puede comprobar cuando hay actualizaciones.

update_url = 'http.....zip'
; opcional Una URL de donde descargar la nueva versión del plugin

require = 'facturacion_base'
; opcional Indica que el plugin facturacion_base es imprescindible para el tuyo.

Crea este archivo dentro de tu plugin y recarga la página.

hola_mundo.php

Ahora vamos a añadir una nueva página, con su entrada en el menú y todo. Simplemente tenemos que crear el directorio controller dentro de nuestro plugin, a continuación creamos un archivo php dentro de ese directorio, por ejemplo hola_mundo.php 

<?php

/// la clase se tiene que llamar igual que el archivo
class hola_mundo extends fs_controller
{
   public function __construct()
   {
      /// se crea una entrada 'Hola mundo' dentro del menú 'hola'
      parent::__construct(__CLASS__, 'Hola mundo', 'hola');
   }
   
   protected function private_core()
   {
      /// desactivamos el motor de plantillas HTML
      $this->template = FALSE;

      echo 'Hola mundo!';
   }
}

Una vez creado este archivo, desactiva y vuelve a activar tu plugin. Verás que ha aparecido el menú hola con la entrada hola mundo.

hola_mundo.html

Ahora vamos a añadir un archivo HTML a nuestro plugin para mostrar todo lo que necesitemos. Pero primero hay que eliminar el código que hemos puesto para desactivar el motor de plantillas HTML: 

<?php

/// la clase se tiene que llamar igual que el archivo
class hola_mundo extends fs_controller
{
   public function __construct()
   {
      /// se crea una entrada 'Hola mundo' dentro del menú 'hola'
      parent::__construct(__CLASS__, 'Hola mundo', 'hola');
   }
   
   protected function private_core()
   {
      /// tu código php lo pondrás aquí
   }
}

Ahora creamos la carpeta view dentro de nuestro plugin, y dentro creamos el archivo hola_mundo.html

Ahora ya puedes continuar con la guía de programación:

  Guía de programación

Este software sigue el patrón MVC. Los modelos los tienes en la carpeta model de cada plugin, los controladores en controller/, las vistas en view/ y el js en view/js/.

El motor de plantillas que usamos para generar el HTML es una versión personalizada de RainTPL. Es un motor muy muy básico, soporta solamente un par de funciones: if, loop, include, function y poco más.

El github del núcleo de FacturaScripts lo tienes aquí: https://github.com/NeoRazorX/facturascripts_2015 El núcleo se encarga de la gestión de usuarios, plugins y poco más.

Los plugins tiene una prioridad, el último plugin activado tiene más prioridad que el anterior. A la hora de buscar un controlador, modelo, vista, etc... FacturaScripts busca primero en el último plugin activado, luego en el anterior, etc... Con esto ya te lo he contado todo, puedes sobreescribir el funcionamiento de cualquier plugin siempre que el tuyo tenga más prioridad.

Toda la parte de facturación y contabilidad está en el plugin facturacion_base: https://github.com/NeoRazorX/facturacion_base
